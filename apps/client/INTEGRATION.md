# Интеграция фронтенда с бекендом

## ✅ Что было сделано

1. **API клиент** (`src/lib/api.ts`) - настроен Axios с базовым URL
2. **React Query хуки**:
   - `use-categories.ts` - для работы с категориями
   - `use-accounts.ts` - для работы со счетами
   - `use-transactions.ts` - для работы с транзакциями
3. **Обновлены все компоненты** для использования API вместо локального store
4. **Настроен QueryClient** с оптимальными параметрами

## 🚀 Запуск проекта

### 1. Запустите бекенд

```bash
cd server
npm install
# Создайте .env файл на основе env.example
cp env.example .env
# Настройте базу данных PostgreSQL в .env
npm run start:dev
```

Бекенд будет доступен на `http://localhost:3000`

### 2. Запустите фронтенд

```bash
# В корне проекта
npm install
npm run dev
```

Фронтенд будет доступен на `http://localhost:8080`

## 📝 Переменные окружения

### Фронтенд
Создайте файл `.env` в корне проекта:
```
VITE_API_URL=http://localhost:3000
```

### Бекенд
Создайте файл `.env` в папке `server` (см. `server/env.example`)

## 🔄 Как это работает

1. **React Query** автоматически кэширует данные и обновляет их при мутациях
2. **Axios** отправляет запросы к бекенду
3. **Toast уведомления** показывают успех/ошибку операций
4. **Автоматическая инвалидация кэша** при создании/обновлении/удалении данных

## 📡 API Endpoints

- `GET /categories` - получить все категории
- `POST /categories` - создать категорию
- `PATCH /categories/:id` - обновить категорию
- `DELETE /categories/:id` - удалить категорию

- `GET /accounts` - получить все счета
- `POST /accounts` - создать счет
- `PATCH /accounts/:id` - обновить счет
- `DELETE /accounts/:id` - удалить счет

- `GET /transactions?type=income` - получить транзакции
- `POST /transactions` - создать транзакцию
- `DELETE /transactions/:id` - удалить транзакцию

## ⚠️ Важно

- Убедитесь, что база данных PostgreSQL запущена и настроена
- CORS настроен для `http://localhost:8080`
- При создании транзакции баланс счета обновляется автоматически на бекенде
